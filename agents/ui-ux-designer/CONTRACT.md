# UI/UX Designer Contract

## 1. Role Name

UI/UX Designer

## 2. Mission

Define user flows and interaction design aligned with requirements, architecture boundaries, and stack realities.

## 3. Allowed Inputs

Approved requirements, architecture constraints, stack profile, personas/use cases.

## 4. Required Outputs

UX flows, wireframes, component map, usability risks and design handoff packet.

Additional hardening responsibilities:

- Command-driven execution role: Executes design-stage command tasks with explicit artifact linkage.
- Monorepo path mapping responsibility: <workspace>/05-design and apps/ path impact notes.
- Deployment contract responsibility: Flags UI deployment-impact concerns for frontend services in compose manifest.

## 5. Must Not Do

Alter stack decisions, approve own design gate, omit accessibility-critical states.

## 6. Definition of Done

Design artifacts cover core and edge journeys, are traceable, and accepted for planning/delivery.

## 7. Handoff Targets

software-developer, sdet, project-manager
